How publishing works

  1. Host the file. In your GitHub repo, open the images folder → Add file → Upload files. This is your permanent, free image host. (Any hosted URL also works — Squarespace CDN links keep working while your subscription is active.)
  2. Point the site at it. Paste the URL below — or images/your-file.jpg if it lives in the repo. The 📁 button previews a file straight from your phone and fills in the repo path for you.
  3. Publish. Tap Generate assets.js, copy the code, and paste it into the assets.js file in your GitHub repo (edit → paste → commit). Vercel redeploys automatically in ~30 seconds.

Files picked with 📁 are previews only — the actual image still needs to be uploaded to the repo's images folder with the same name.

Videos: use short, compressed .mp4 clips (aim for under ~10 MB and 30 seconds). They play muted and looped. For anything longer, use the YouTube slot instead — it keeps the site fast.
